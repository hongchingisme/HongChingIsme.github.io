2. gird用法

這裡是一種我從沒接觸過的切版方式，讓會讓包住的div自動去切成各符合大小的容器，非常方便。

專案很多都是使用 display: grid; grid-template-columns:repeat(3,1fr); grid-gap: 2rem;

而 gird的用法 都要先從 display:grid 開始，就可以自定義了，接著用 grid-template-columns:repeat 輸入重複次數與切法，像這邊的 fr 就是 「 全名是 fraction (分數)，這個分數不是 score 那個分數，而是數學分子 + 分母的那個分數。

我們可以用 fr 去設定 CSS Grid 的相對寬度，以達到 RWD 的縮放效果。

這邊輸入的1fr 就是 1/3的意思，將版面切成三個 3/1 就會變成各三個的意思，非常方便。

最後用grid-gap 去調整 上下間距。

4. 選單選取後，定位與圖層的觀念

這邊是我搞不懂的地方，絕對定位跟相對定位我有基礎的知識，例如沒有設定定位點就會自動定位在 body 上，但我不懂的是，為什麼要設定兩個相對定位呢?

這邊是要實現一個功能，就是當我滑鼠移過去的時候，我的底部會有一個背景色跑出來的效果，

專案教學是再寫一個 div ， 給他大小與我們要的背景漸層顏色，然後偏移，使用定位，定在盒子上，接著用用z-index的方式讓圖層往下，再寫一個 hover 效果就出來了。

但是很明顯沒那麼簡單，實際上影片也是有遇到圖層設定不好的問題。

教學影片-17:37開始

後來是在 文字盒子上也加上一個相對定位就成功了，我不太會用文字說明，用圖片來說好了：

我後來找一些定位的文章：

找到了一句話　：　

z-index屬性設置了一個定位元素及其後代元素或flex項目的z-order。

我頓悟了。

原來要在將有覆蓋到的 div 都要給予定位，才可以進行圖層排序，後來問題就解開了。

5. 不會將網頁寫成響應式

因為我沒有寫成響應式的經驗，一開始就是直接看直接切，因為不想影片寫一行我寫一行，這樣很像沒有在思考。

但後來發現我都用　px　當單位在切版，到響應式會跑掉，所以後來就直接重作，並且跟著影片打程式碼，然後了解寫法背後的原因。

我自認幾個響應式要注意的地方：

  1. 單位要統一　例如：文字用rem
  2. 寬高用百分比
  3. 區塊可以用padding去撐高度

這樣就比較好來製作響應式的網站
