# Git 基礎

什麼是Git?

Git 就是「版本控制」，也就是存檔、讀檔的意思，當我們在程式開發的時候，一定會遇到不小心改錯檔案，或者要保留原本檔案另開一個版本出來加入功能測試的時候。

例如遊玩超級瑪利歐的時候，瑪利歐被烏龜打死，死掉會自動回到上一個復活點，但要復活之前，要有個存檔點，如果沒有使用版本控制，當不小心刪除檔案時，就真的可以幫檔案辦理告別式了。

(存檔案例)

完成之後可以看見資料夾的背後多出了(master)。

加入成功之後可以透過指令 git status 來觀察檔案狀態。

上圖顯示新增了一個新的文件: hello.html 代表新增成功。

接著，輸入git commit -m "add new file hello.html進行存檔。

我們再次輸入git status來查看狀態。

就可以發現我們已經成功存檔完畢！

接著下載 Source Tree 。

點我到官網下載

打開 Soure Tree 可以看見剛剛的存檔點已經在上面了。

接著我們透過 VScode 打開新增的檔案 hello.html 來修改裡面的內容，接著然後輸入 git status 查看狀態。

就會發現已經出現紅字，告訴你說檔案被修改了。

接著我們可以輸入 git commit -m "modify hello.txt" 儲存修改這個檔案。

什麼是 Github?

Github是一個可以將自己的作品上傳上去的網站，在上面有很多人分享自己的專案、程式碼內容去讓大家學習，是一個社群網站，我們可以透過Github上傳我們的作品，也可以將別人的作品下載下來參考與學習。

產生 ssh key 綁定電腦與Github帳號吧！

在上傳作品到 Github 之前，要先綁定 ssh key，那什麼是 ssh key 呢?

ssh key 可以理解成鑰匙鎖，電腦與 Github 之間需要一組帳號密碼，透過綁定可以達到免密碼登入的模式。

綁定ssh key之後，就可以開始在Github中上傳檔案了!

(因為我已經綁定完成了，所以這邊用敘述的方式)

  • 首先，我們先輸入:

ssh-keygen

  • 接著電腦會詢問ssh key要存放的位置，可以設定為預設的地方:

Generating public/private rsa key pair. Enter file in which to save the key (/home/username/.ssh/id_rsa):

  • 接著會詢問是否設定密碼，可以按空白表示不設定。

Enter passphrase (empty for no passphrase): Enter same passphrase again:

然後就會產生一組 ssh key了!

  • 接著複製你的 ssh key

輸入指令:

ssh-copy-id -i your_key_path username@server_host

然後輸入:

cat ~/.ssh/ubuntu_id_rsa.pub

會顯示公鑰的資料複製下來。

接著我們到 Github 中點擊設定:

並找到SSH and GPS keys ，進入後點擊右上角的 New SSH key

開始新增 ssh key

新增成功之後可以看到 ssh key 已經有剛剛新增好的欄位了!
