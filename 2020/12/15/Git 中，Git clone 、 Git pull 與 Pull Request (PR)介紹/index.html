git 中，git clone 、 git pull 與 Pull Request (PR)介紹

git clone

在 git 當中，我們可以透過git clone 指令來達到將專案倉儲克隆一份，就好比把檔案給複製到自己的電腦當中，而克隆的方法也很簡單，以下的步驟會告訴你怎麼做：

  1. 第一步，進入到想要克隆的專案中

  1. 進入主畫面　code 當中，並點取右上角的：

  1. 點入後會看見 Clone 的字樣，下面有顯示不同的克隆方法，這邊介紹使用 SSH 克隆的辦法，因為我已經與 github 綁定好了 ssh key ，可以直接透過 cmd 輸入指令來客隆

  2. 將 ssh 當中的指令複製下來，並開啟　cmd

  1. 在 cmd 當中輸入 git clone + 複製好的指令

輸入後就會克隆成功了！

:::info
在克隆之前一定要知道自己克隆在哪邊，克隆之後會直接為一個資料夾，名稱為專案的名字。
:::

git pull

當我們準備將新的檔案更新至本地端時，就會用到 git pull 指令，藉著 pull 命令，可以把遠端數據庫修改的內容合併到本地端數據庫。pull = fetch + merge」。

而使用 git pull 會直接與本地的數據 merge 要特別注意。

Pull Request 流程

當我們要與別人進行專案協作時，假如這個專案並不是我認識的朋友，而是一個路人，那要如何跟他在 github 合作呢?

很簡單

使用 Pull Request 就可以與他人合作。

首先，我們透過 fork 將你感興趣的專案給複製一分到自己的 github 帳號當中。

點擊下去之後可以告訴原作者說你新增了什麼功能，若原作者喜歡這個功能就會將你的 PR 給 合併(marge)進去他的專案當中。

為什麼需要透過 Pull Request 來開發?

因為在合作開發專案的時候，每個人負責的功能與部分會不同，當我們要同時對一個專案進行開發時，就要透過建立主要分支與其他分支來達成開發的效果。

主要分支

為此專案主要使用的版本，當在其他分支中經過審核沒問題後，才會將他與主要分支進行合併，

其他分支

一個專案當中，每個人被分派到的任務不一樣，所以我們可以先創立一個主要分支外的分支，將新增個功能與修改審查後合併在這個分支上，沒有問題的話，再與主要分支合併。

因為每間公司對於 git 的合作方式並不一樣，所以還是要依照自己公司開發流程為主。

換一種方式來說 PR :

有一本書，這本書很特別，要求每個人都要在書中創造自己的故事，但故事的內容需要審查，不能腥羶色所以有一個主要在審查內容的管理者，而這個管理者的電腦當中有三個資料夾:

  1. 確定出版的內容資料夾 (主要分支)
  2. 二審的內容資料夾 (其他分支 - 全部的測試通過才可以進入主要分支，並當作主要分支的版本控制)
  3. 初審的內容資料夾 (其他分支 - 基本功能測試OK就合併)

每個作者寫完作品之後，都要先透過管理者的審查，初審時，管理者認為通過，就會放入第三個資料夾，二審時拿給總編輯長閱讀後通過才進入到第一個資料夾。

而第三個資料夾是可以隨時修改的，方便讓初審就被駁回的作者可以重新在更改要求審查。
