# Git 基礎

Git　建立分支、合併分支、分支檔案衝突與　Git 指令彙整 與 vim 編輯模式

建立分支 (Branch)

  • 首先我們先建立一個檔案 hello_branch.html。

  • 然後輸入git add hello_branch.html。

  • 接著輸入git status觀察狀態。

  • 建立一個分支:

git branch dev 這裡建立了一個名為 「dev」的分支。

然後輸入 git btanch 來查看目前有的分支。

可以看到已經新增了一個名為「dev」的分支。

  • git checkout 來切換分支

這邊輸入 git checkout dev 進入剛剛創建好的 「dev」 分支

  • 接著，我們將剛剛創建好的檔案hello_branch.html 輸入:git commit -m "hey branch" 進行存檔

存檔輸入ls來查看各分支的內容

可以發現，因為我們是在 dev 的支線當中去儲存新增的hello_branch.html檔案，也就是說這個檔案不會出現在原本的 master 支線裡面，可以想像成，遊戲當中存檔也是如此，不同的分支，可以放入不同的劇情存檔:

存檔內容會隨著進度的推移與角色變強來有所不同

分支如何 Merge與衝突解決 Conflict

  • 我們在剛剛的 master 分支中也新增一個一模一樣的檔案hello_branch.html，並且儲存與存檔

  • 接著，我們將兩個分支給合併，故意製造衝突

輸入: git merge dev 將 master 與 dev 進行合併

也可以在 Source Tree 當中看到分支已經被合併

接著就會發生衝突:

因為兩個分支裡面都有hello_branch.html這個檔案，而檔案內的內容有不一樣的地方，系統會將兩者都顯示出來，你可以選擇你想保留的部分即可！

Git 指令彙整

git init :建立資料庫

git add . : 新增檔案

git commit -m "add . " 新增存檔

git branch : 查看分支

git branch . 建立分支

git checkout . 進入分支

git merge . 合併分支

git commit -m "add . " 新增存檔 這個指令當不輸入 -m 時候會進入 vim 編輯模式

在這個 vim 編輯模式當中 有特殊的指令用法

i : 輸入i才可以輸入文字

:q : 離開

:wq:儲存離開

:q! : 忽略修改直接離開
